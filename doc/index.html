Možnosti testování

Javascriptu

a jejich krátké srovnání

@danielmilde / Medio Interactive

O čem to dnes bude?

  • Mocha
  • PhantomJS
  • JSTestDriver
  • Testacular

Mocha

  • testy se vykonávají přímo v Nodejs
  • potřeba mít unitové testy || simulovat DOM API
nehodí se pro testování kompatibility prohlížečů

PhantomJS

  • spouští se "prohlížeč" PhantomJS obsahující WebKit jádro
  • potřeba udržovat HTML stránku, kde se spustí testy

JsTestDriver

  • spouští se javovská binárka
  • na začátku je potřeba nastartovat server (a prohlížeče)
  • je potřeba separátní konfigurační skript
  • podpora téměř všech prohlížečů

Testacular

  • spouští se webový server v Nodejs a prohlížeč
  • je potřeba separátní konfigurační skript
  • podpora všech mainstream prohlížečů
  • ranná fáze projektu
prohlížeč komunikuje se serverem pomocí Socket.IO
umožnuje vše spustit a pak zase hned zavřít
